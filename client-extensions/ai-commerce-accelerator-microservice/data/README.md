# Mock Data Configuration

This directory contains JSON configuration files that allow you to customize the mock data generated by the AI Generator without modifying the source code.

## Configuration Files

### `categories.json`

Defines product categories, their available product names, options (like color, size), and specification types.

**Structure:**

```json
{
  "CategoryName": {
    "names": ["Product Name 1", "Product Name 2", ...],
    "options": [
      { "name": "Option Name", "values": ["Value1", "Value2", ...] }
    ],
    "specs": ["Spec1", "Spec2", ...]
  }
}
```

### `specifications.json`

Defines the possible values for each specification type per category.

**Structure:**

```json
{
  "CategoryName": {
    "spec-key": ["Value1", "Value2", "Value3", ...]
  }
}
```

### `pricing.json`

Defines pricing ranges and modifiers for different categories and options.

**Structure:**

```json
{
  "CategoryName": {
    "basePrice": { "min": 50, "max": 500 },
    "priceModifiers": {
      "OptionName": { "OptionValue": modifier_amount }
    }
  }
}
```

## Customization

You can modify these files to:

- Add new product categories
- Change product names and descriptions
- Modify available options and their values
- Adjust specification types and values
- Set custom pricing ranges and modifiers

## Multi-Language Support

The system automatically generates multi-language versions of all text content based on the selected languages in the configuration. The JSON files define the base (English) content, and the system appends language suffixes for other languages.

## Hot Reload

Changes to these files require restarting the microservice to take effect. In the future, a hot reload feature may be added.

## Fallback Behavior

If any configuration file is missing or contains errors, the system will fall back to hardcoded default values to ensure continued operation.
